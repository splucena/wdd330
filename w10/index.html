Client Side Form Validation

Client-side validation is an initial check and an important feature of good user experience; by catching invalid data on the client-side, the user can fix it right away.

Client-side validation should not be considered an exhaustive security measure because client-side validation is easy to bypass. Validation done in the browser is called client-side validation, while validation done on the server is called server-side validation.

Reasons for Client-side Validation

  • Rigth data, right format
  • Protect users' data
  • Protect ourselves

Different Types of Client-side Validation

  • Built-in form validation (HTML5)
  • JavaScript

Using Built-in Form Validation

  • required: field needs to be filled in befor the form can be submitted
  • minlength and maxlength: minumum and maximum text length
  • min and max: minimum and maximum values of numerical input type
  • type: specifies whether the data needs to be a number, an email, an email address, or some other specific type
  • pattern: regular expression that defines the pattern the entered data needs to follow

CSS Pseudo-class

  • :valid
  • :invalid
  • :out-of-range

Validating Form Using JavaScript

  • patternMismatch: returns true if value does not match the specified pattern
  • tooLong: returns true if value is longer than specified maxlength
  • tooShort: returns true if value is shorter than specified minlength
  • rangeOverFlow: returns true if the value is greater than the maximum specified by the max attribute
  • rangeUnderflow: returns true if the value is less than the minimum specified by the min attribute
  • typeMismatch: returns true if the value is not in the required syntax (when type is email or url)
  • valid: returns true if the element meets all its validation constraints
  • valueMissing: returns true if the element have a required attribute, but no value, or false otherwise
