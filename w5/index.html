Testing And Debugging

Errors, Exceptions, and Warnings

Errors are caused when something goes wrong in a program and are usually caused by the following:

  • System error - problem with the system or external device with which the program is interacting
  • Programmer error - incorrect syntax or faulty logic
  • User error - incorrect data

Exceptions

An error that produces a return value that can be used by the programmer to deal with the error

Stack Traces

An exception will also produce a stack trace which a sequence of functions or method calls that can lead to where the error occurred.

Warnings

Errors that doesn't cause the program to crash but may cause the program to run incorrectly.

Throwing Exceptions

Programmers can also throw exception instead of JavaScript throwing it automatically in the program

Test-driven Development

The process of writing tests before any actual code.

  1. Write tests (that initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features

Testing Frameworks

Provides a structure to write and run tests like Jest.
